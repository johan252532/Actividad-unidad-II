1- ¿Cuáles son las principales funciones de un kernel en la arquitectura de un sistema operativo y cómo interactúa con el hardware?

El kernel es el núcleo del sistema operativo y sus principales funciones son gestionar el CPU, la memoria, los dispositivos (a través de drivers) y las llamadas al sistema. Interactúa directamente con el hardware a través de los controladores, actuando como un intermediario que traduce las peticiones de las aplicaciones a comandos que el hardware puede entender.

2- ¿cómo se gestionan los procesos y cuál es la importancia de la multitarea en la eficiencia del sistema?

La gestión de procesos implica que el sistema operativo crea, planifica, ejecuta y finaliza programas (procesos). Asigna recursos como la CPU y la memoria a cada proceso y los alterna rápidamente para dar la ilusión de ejecución simultánea. La multitarea es crucial para la eficiencia del sistema porque permite ejecutar múltiples tareas concurrentemente, optimizando el uso de los recursos del sistema (especialmente la CPU) y mejorando la capacidad de respuesta para el usuario al no tener que esperar a que una tarea finalice para iniciar otra.

3- ¿Qué medidas de seguridad se pueden implementar a nivel de sistema operativo para proteger una red, especialmente en entornos empresariales?

A nivel de sistema operativo, para proteger una red empresarial se pueden implementar medidas como: Gestión de usuarios y autenticación robusta: Contraseñas seguras, políticas de complejidad y autenticación multifactor. Control de acceso basado en roles (RBAC): Asignar permisos específicos a usuarios y grupos según sus funciones. Firewall integrado: Filtrar el tráfico de red entrante y saliente, bloqueando conexiones no autorizadas. Políticas de seguridad: Configurar directivas para contraseñas, bloqueo de cuentas y auditoría. Actualizaciones y parches de seguridad: Mantener el sistema operativo y sus componentes actualizados para corregir vulnerabilidades. Auditoría y registro de eventos: Monitorear la actividad del sistema para detectar y analizar posibles incidentes de seguridad. Cifrado de datos: Proteger la información sensible tanto en tránsito como en reposo. Virtualización y aislamiento: Separar entornos para limitar el impacto de posibles fallos de seguridad.
